# 文章标题

## 一句话摘要

用 2 到 3 句话说明这篇文章解决什么问题、适合谁看、结论是什么。

## 背景

- 当前场景
- 关键约束
- 为什么值得记录

## 结论先行

!!! note "结论"
    把最重要的判断写在这里，方便快速浏览。

## 正文

### 方案或过程

按时间线、问题拆解或模块结构组织都可以，但保持同一篇文章内的组织方式一致。

### 关键代码或配置

```python
def example() -> str:
    return "replace me"
```

### 对比或补充说明

=== "推荐做法"

    写推荐方案、适用条件和边界。

=== "备选做法"

    写备选方案、缺点和不选它的原因。

## 踩坑

- 记录容易忽略的问题
- 记录失败路径和修正方式

## 参考

- 放外部链接、文档或资料来源
