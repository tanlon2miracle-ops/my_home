# 本地写作与发布工作流

这篇示例文章用于固定首版的日常操作习惯。

## 工作流

1. 新建文章目录和 `index.md`
2. 激活 `.venv`
3. 本地运行 `mkdocs serve`
4. 提交并推送到 `main`

## 为什么不手工发布

=== "当前方案"

    只负责写作和提交，构建与部署交给 GitHub Actions。

=== "不采用的方案"

    不使用 `gh-pages` 分支，避免本地和远端同时维护两套发布状态。
